🖥️ WAMP Server Setup (Step-by-Step)

STEP 1

Download & Install WAMP

  1. Go to wampserver.com and download the installer for your OS (32-bit or 64-bit).
  2. Run the installer as Administrator.
  3. Follow the wizard — use all default settings.
  4. WAMP installs Apache (web server), MySQL (database), and PHP together.
STEP 2

Start WAMP Server

  1. Open WAMP from your desktop or Start menu.
  2. The WAMP icon appears in the system tray (bottom-right of screen).
  3. Wait for the icon to turn GREEN — this means Apache and MySQL are both running.
  4. If the icon is orange, click it → hover "Apache" → click "Start/Resume Service".
💡 Tip: If port 80 is blocked (icon stays orange), another program (like Skype or IIS) may be using it. Right-click WAMP tray icon → Apache → "httpd.conf" → change "Listen 80" to "Listen 8080". Then access your project at http://localhost:8080/cse_repository/
STEP 3

Place Project in www Directory

  1. Open File Explorer.
  2. Navigate to: C:\wamp64\www\ (or C:\wamp\www\ on 32-bit)
  3. Copy your entire cse_repository folder into this www directory.
  4. Final path should be: C:\wamp64\www\cse_repository\
📌 Why www? WAMP's Apache server serves files from the www folder. Everything inside www becomes accessible at http://localhost/.
STEP 4

Create the Database in phpMyAdmin

  1. Open your browser and go to: http://localhost/phpmyadmin
  2. Login: Username = root, Password = leave blank (default)
  3. Click the "SQL" tab at the top.
  4. Copy the entire contents of database.sql and paste it into the SQL box.
  5. Click "Go" — this creates the database, table, and sample data.

Alternative — Import the SQL file:

  1. In phpMyAdmin, click "New" in the left sidebar.
  2. Type cse_repository as the database name → click Create.
  3. Click on cse_repository in the sidebar.
  4. Click the "Import" tab → "Choose File" → select database.sql → click Go.
STEP 5

Run the Project

  1. Make sure WAMP icon is GREEN.
  2. Open your browser.
  3. Type: http://localhost/cse_repository/
  4. The home page should load with sample projects!
✅ Quick Checklist: WAMP is green ✓ · Folder is in www ✓ · Database created ✓ · db.php credentials match ✓

🎓 Viva Questions & Answers

Section A — PHP & MySQL Basics

Q1. What is PHP? Why did you use it in this project?
PHP (Hypertext Preprocessor) is a server-side scripting language used to build dynamic web pages. We used it because PHP can connect to MySQL databases, process form data, and generate HTML dynamically — all without using complex frameworks. It runs on the WAMP server's Apache, making it perfect for beginner projects.
Q2. What is WAMP Server? What does each letter stand for?
WAMP stands for: Windows, Apache, MySQL, PHP. It is a local development environment that installs all three tools together on Windows. Apache is the web server, MySQL is the database, and PHP is the scripting language. WAMP lets us run PHP websites on our own computer without needing to buy hosting.
Q3. How do you connect PHP to MySQL?
We use the mysqli_connect() function: 
$conn = mysqli_connect("localhost", "root", "", "cse_repository");
The four parameters are: host, username, password, and database name. We then check if (!$conn) and call mysqli_connect_error() to show the error if it fails. We put this in a separate file called db.php and include it in every PHP page.
Q4. What is the difference between GET and POST methods?
GET sends data in the URL (visible to the user), e.g., project-detail.php?id=3. It is used for reading data. POST sends data in the request body (not visible in URL). It is used for submitting forms, inserting, updating, or deleting data. In our project, we use GET to pass the project ID in URLs, and POST for all form submissions (upload, edit, delete).
Q5. What is AUTO_INCREMENT in MySQL?
AUTO_INCREMENT automatically assigns a unique increasing number to a column each time a new row is inserted. We use it for the id column so every project gets a unique ID (1, 2, 3, ...) without us manually typing it. It is always used with PRIMARY KEY.
Q6. What is PRIMARY KEY?
A PRIMARY KEY is a column (or set of columns) that uniquely identifies each row in a table. No two rows can have the same PRIMARY KEY value, and it cannot be NULL. In our project, id is the PRIMARY KEY of the projects table. We use it to fetch, update, or delete a specific project.
Q7. What does mysqli_fetch_assoc() do?
mysqli_fetch_assoc() fetches one row from a query result as an associative array. This means you can access column values by their column name, e.g., $row['title'] or $row['student_name']. We use it inside a while loop to go through all results one by one: 
while ($row = mysqli_fetch_assoc($result)) { echo $row['title']; }
Q8. What is SQL injection? How did you prevent it?
SQL injection is a security attack where a hacker enters malicious SQL code into a form field, which gets executed in the database. For example, entering ' OR 1=1 -- in a login form could bypass authentication. In our project, we prevent it using mysqli_real_escape_string($conn, $input) which escapes dangerous characters like quotes before inserting them into a query. We also cast the ID to integer using (int)$_GET['id'].
Q9. What does htmlspecialchars() do?
htmlspecialchars() converts special HTML characters to their safe equivalents so they display as text and cannot be executed as HTML or JavaScript. For example, < becomes &lt; and > becomes &gt;. We use it when displaying data from the database to prevent XSS (Cross-Site Scripting) attacks.
Q10. Explain the INSERT query used in your project.
INSERT INTO projects 
  (title, student_name, year, category, technology, description, github_link)
VALUES 
  ('$title', '$student_name', $year, '$category', '$technology', '$description', '$github_link')
INSERT INTO specifies the table name. The column names in parentheses tell MySQL which columns to fill. VALUES contains the actual data. We do not include id or created_at because AUTO_INCREMENT and DEFAULT CURRENT_TIMESTAMP fill those automatically.
Q11. Explain the UPDATE query used in your project.
UPDATE projects SET
  title = '$title',
  student_name = '$student_name',
  year = $year
WHERE id = $id
UPDATE modifies existing records. SET specifies which columns to change and their new values. WHERE id = $id is critical — it restricts the update to only the specific project. Without WHERE, ALL rows in the table would be updated!
Q12. Explain the DELETE query used in your project.
DELETE FROM projects WHERE id = $id
DELETE FROM removes rows from a table. WHERE id = $id ensures only the specific project with that ID is deleted. Without the WHERE clause, the entire table would be emptied! We also use mysqli_affected_rows() to confirm that at least one row was actually deleted.

Section B — JavaScript Validation

Q13. Why do you need JavaScript validation if PHP also validates?
JavaScript validation happens on the client side (in the browser, before the form is sent). This gives instant feedback to the user without a page reload — it's faster and better UX. PHP validation happens on the server side (after the form is submitted). We need both because JavaScript can be disabled by the user, so PHP validation is the safety net. The rule is: "Never trust user input — always validate on the server."
Q14. How does your JavaScript search and filter work?
All projects are loaded into the page as HTML cards. When the user types in the search box or selects a category, an event listener calls the filterProjects() function. This function reads the search term and selected category, then loops through every .project-card element and checks if its content matches. If it matches, the card stays visible (display: flex); if not, it is hidden (display: none). This happens instantly without any database queries.
Q15. What is confirm() in JavaScript and where did you use it?
confirm() is a built-in JavaScript function that shows a popup dialog with a message and OK / Cancel buttons. It returns true if the user clicks OK, and false if they click Cancel. We use it in the delete button's onsubmit attribute: onsubmit="return confirmDelete('Project Name')" If the user cancels, false is returned and the form is NOT submitted, preventing accidental deletion.

Section C — Project Architecture & Design

Q16. What is CRUD? Give examples from your project.
CRUD stands for Create, Read, Update, Delete — the four basic database operations:
• Create: upload.php → INSERT query → adds a new project
• Read: index.php / projects.php / project-detail.php → SELECT query → fetches projects
• Update: edit-project.php → UPDATE query → modifies a project
• Delete: delete-project.php → DELETE query → removes a project
Q17. Why did you create a separate db.php file?
We separated database credentials into db.php to follow the DRY principle (Don't Repeat Yourself). Instead of writing mysqli_connect() in every file, we write it once and use include 'db.php'; in each PHP file that needs it. If we ever change the database name or password, we only need to update one file.
Q18. What is the purpose of htmlspecialchars() vs mysqli_real_escape_string()?
mysqli_real_escape_string() — used when inserting data INTO the database. It escapes SQL special characters (like quotes) to prevent SQL injection.

htmlspecialchars() — used when outputting data FROM the database to HTML. It prevents XSS attacks by converting HTML characters to safe entities so malicious scripts cannot run.

Both are needed: one protects the database, the other protects the browser.
Q19. How does project-detail.php know which project to show?
The project ID is passed in the URL as a GET parameter: project-detail.php?id=3. PHP reads it using $_GET['id']. We first validate that it exists and is numeric using isset() and is_numeric(), then cast it to integer with (int) to prevent injection. Then we run SELECT * FROM projects WHERE id = $id to fetch that specific project's data.
Q20. What is the difference between include and require in PHP?
Both include and require insert the contents of one PHP file into another. The difference is in error handling: if the file is not found, include gives a warning and the script continues running, while require gives a fatal error and stops the script. We use include 'db.php' — since without the database connection, the page will fail anyway, either include or require would work here.
Q21. What is the purpose of the hidden input field in the edit and delete forms?
Hidden input fields (<input type="hidden">) send data with the form that the user cannot see or change in the form UI. In our project:
• In edit-project.php: <input type="hidden" name="id" value="3"> passes the project ID to the POST handler so PHP knows which project to UPDATE.
• In delete-project.php: <input type="hidden" name="redirect" value="projects.php"> tells the delete handler where to redirect after deletion.
Q22. What is ORDER BY and LIMIT in SQL?
ORDER BY sorts query results by a column. ORDER BY created_at DESC sorts newest projects first (DESC = descending, Z→A or newest→oldest). ASC would be ascending (oldest→newest).

LIMIT restricts how many rows are returned. LIMIT 6 returns only the first 6 rows. On the home page, we use SELECT * FROM projects ORDER BY created_at DESC LIMIT 6 to show only the 6 most recent projects.
Q23. How would you add a search feature using PHP + SQL instead of JavaScript?
Instead of filtering with JavaScript, we could pass the search term via GET and use SQL LIKE: 
$search = mysqli_real_escape_string($conn, $_GET['search']);
$query = "SELECT * FROM projects 
          WHERE title LIKE '%$search%' 
          OR student_name LIKE '%$search%' 
          OR technology LIKE '%$search%'";
The % is a wildcard — it matches any characters before or after the search term. Our current JavaScript approach is faster because it doesn't need a server request.
Q24. What improvements would you make if you had more time?
1. Prepared Statements — use mysqli_prepare() for better SQL injection protection.
2. User Authentication — login system so only the project owner can edit/delete their project.
3. File Upload — allow students to upload a project report PDF or screenshot.
4. Pagination — show 10 projects per page instead of all at once.
5. Search on Server Side — SQL LIKE search for better performance with large data.
